# Типы меню

Тип задаётся ключом `type` на верхнем уровне YAML. Значения: `chest` (по умолчанию) и `inventory`.

```yaml
type: chest        # или inventory; любое другое/пустое значение = chest
```

## Настройки верхнего уровня (общие)

| Ключ | Назначение |
|---|---|
| `type` | `chest` или `inventory`. По умолчанию `chest`. |
| `title` | Заголовок окна (для `chest`). Поддерживает цвета и плейсхолдеры. |
| `rows` | Число рядов (только `chest`, 1–6; вне диапазона обрезается). Для `inventory` игнорируется. |
| `commands` | Список команд, открывающих меню (см. [Commands-and-Permissions](Commands-and-Permissions.md)). |
| `triggers` | Легаси-алиас `commands` (списки объединяются, приводятся к нижнему регистру, дублируются). |
| `command-description` | Описание зарегистрированной команды меню (для `/help`). |
| `show-in-help` | Показывать команду меню в `/help`. По умолчанию `true`. |
| `permission` | Право на открытие меню (на всех путях). Пусто = без ограничения. |
| `open-requirement` | Условие открытия меню (см. [Requirements](Requirements.md)). |
| `items` | Предметы по слотам (см. [Items-and-Clicks](Items-and-Clicks.md)). |
| `update-interval` | Живое обновление плейсхолдеров открытого меню, тики (см. [Advanced-Menu-Options](Advanced-Menu-Options.md)). |
| `args` | Имена аргументов команды открытия → `{имя}`/`{argN}`/`{args}` (см. [Advanced-Menu-Options](Advanced-Menu-Options.md)). |
| `open-item` | Предмет-открывашка в хотбаре, ПКМ открывает меню (см. [Advanced-Menu-Options](Advanced-Menu-Options.md)). |
| `open-animation` | Пошаговое появление предметов при открытии, только `chest` (см. [Advanced-Menu-Options](Advanced-Menu-Options.md)). |

## `type: chest`

Сундук-GUI через **InvUI** (шейдится в плагин, отдельно ставить не надо).

- `rows: 1–6` — число рядов. Итоговый размер сетки = `rows × 9` слотов.
- **Слоты** нумеруются от `0` до `rows*9 - 1`, слева-направо, сверху-вниз. Для 3 рядов — слоты `0..26`.
- Заголовок берётся из `title`, разворачивает плейсхолдеры под конкретного игрока при открытии.
- Навигация (`open_menu`/`back`/`refresh`) и история переходов работают через `MenuService`.

```yaml
type: chest
title: "<gold>Главное меню"
rows: 6                 # 54 слота (0..53)
items:
  '0': { material: ARROW, name: "<yellow>Назад", clicks: { any: [ { type: back } ] } }
```

## `type: inventory` (экспериментально)

Меню рендерится **прямо в инвентарь игрока**: основные 27 слотов становятся поверхностью меню, а хотбар,
крафт и броня остаются обычными.

Включение — в `config.yml`:

```yaml
inventory-menu:
  enabled: false      # true — показывать меню-инвентарь всем при входе
  default: ""         # id меню (type: inventory), которое показывать по умолчанию
```

Ключевые факты:

- **Нумерация слотов в YAML — `0..26`**, а НЕ 9–35. Физически они ложатся в основные слоты инвентаря
  (9–35): YAML-слот `0` → слот инвентаря `9`, YAML-слот `26` → слот `35`. То есть в файле пиши `items.0 …
  items.26`, хотя визуально это верхние три ряда инвентаря.
- `rows` для inventory-меню **игнорируется** (всегда 27 ячеек).
- Меню-по-умолчанию (`inventory-menu.default`) авто-показывается при входе/респавне — но **тихо** уважает
  `permission` и `open-requirement` меню (без спама deny-сообщений всем).
- Переключение между inventory-меню — действием `open_menu` (см. [Actions](Actions.md)).
- Отключить конкретному игроку: `/am invclose [игрок]` (вернётся при `/am open <inv-меню>` или перезаходе).

⚠️ **Экспериментально / безопасность:** пока меню активно, слоты 9–35 — «только меню»: положить туда вещи
нельзя, клики/перетаскивания в этой зоне отменяются. При **первом** показе любые реальные вещи из этих
слотов **выпадают игроку** (никогда не удаляются). На выходе с сервера пункты меню снимаются, чтобы не
попасть в сохранённый инвентарь. Тестируй без ценных вещей.

```yaml
type: inventory
title: "HUD"            # title для inventory не показывается как заголовок окна
items:
  '0':                  # физически слот 9 инвентаря
    material: COMPASS
    name: "<green>Спавн"
    clicks: { any: [ { type: run_command, command: "spawn" } ] }
  '26':                 # физически слот 35 инвентаря
    material: NETHER_STAR
    name: "<gold>Меню"
    clicks: { any: [ { type: open_menu, menu: main } ] }
```

## Цвета (`title` / `name` / `lore`)

Форматы можно **смешивать в одной строке** (`util/Text.java`). Легаси-коды переписываются в MiniMessage
**до** разбора, поэтому всё работает вперемешку:

| Формат | Пример | Результат |
|---|---|---|
| Легаси-код | `&c&lПривет` | красный жирный |
| Hex (короткий) | `&#ff8800Привет` | цвет `#ff8800` |
| Bungee/Spigot hex | `&x&f&f&8&8&0&0Привет` | цвет `#ff8800` |
| MiniMessage | `<red>Привет`, `<gradient:#f00:#00f>Радуга</gradient>` | как в MiniMessage |
| Смешанно | `&e<bold>Жёлтый жирный</bold> &#00ff00зелёный` | комбинация |

Легаси-коды: `&0..&9`, `&a..&f` (регистр не важен) — цвета; `&k/&l/&m/&n/&o/&r` —
обфускация/жирный/зачёркнутый/подчёркнутый/курсив/сброс. `&`, за которым нет валидного кода, остаётся как
есть. У **имён и лора предметов** курсив по умолчанию **выключен** (ванильный италик не навязывается).
Полный разбор форматов — на странице [Items-and-Clicks](Items-and-Clicks.md) и в
[Placeholders-and-Integrations](Placeholders-and-Integrations.md).
