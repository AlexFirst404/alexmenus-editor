# Веб-редактор

Веб-редактор AlexMenus сделан **в стиле LuckPerms**: на игровом сервере **не открывается ни один порт**.
Плагин делает только **исходящие HTTPS-запросы** к маленькому paste-сервису (Cloudflare Worker), а правишь
меню в **хостед**-редакторе (статическая страница на GitHub Pages).

## Как это работает

```
/am editor
   → плагин собирает YAML всех меню в бандл
   → POST на paste-сервис  → получает короткий код
   → печатает ссылку  <editor-url>#<код>   (кликабельна в чате)

(открываешь ссылку, правишь меню мышкой в браузере, «Сохранить» → редактор выдаёт код)

/am apply <код>
   → GET бандла с paste-сервиса по коду
   → пишет каждый menus/<id>.yml
   → перезагружает плагин
```

Детали (`bridge/EditorBridge.java`, `command/AlexMenusCommand.java`):

- **Бандл**: JSON `{"v":1,"menus":[{"id":..,"yaml":..}, …]}` — сырой YAML каждого меню.
- **Ссылка** — чистая `<editor>#<код>`. Адрес воркера в ссылку **не попадает** (он вшит в хостед-редактор),
  так что о paste-хосте ничего не утекает.
- Сетевые операции (`push`/`fetch`) идут **асинхронно** (не вешают основной поток); запись файлов и reload —
  в основном потоке.
- `apply` пишет в существующий файл меню, если id уже известен; для **нового** id — в `menus/<id>.yml`
  (в корень категории `general`). Id санитизируется: остаются только `a-z 0-9 _ -`.
- Код (paste) живёт ограниченное время — **24 часа** (по умолчанию у публичного воркера); истёкший код на
  `/am apply` даёт «код не найден или истёк».

## Возможности редактора

- **Мультивыделение слотов** и **контекстное меню** (ПКМ) для быстрых операций.
- **Иконки как в игре** — модель-осознанные превью, блоки рисуются 3D-кубом.
- **Панели requirements** — `view` / `click` / `open` без ручного YAML; композиты (`all`/`any`/`not`) — через
  режим «Расширенно».
- **Вид «Граф»** — кто из какого меню куда ведёт (по действиям `open_menu`).
- **Панель «Настройки меню»** — тип, заголовок, ряды, команды, права, open-requirement.
- **Сырой YAML** — режим прямого редактирования текста меню.

## Настройка (zero-config)

Из коробки настраивать **ничего не нужно**: когда `editor.worker-url` пуст, плагин использует **общий
публичный воркер**, а `editor.url` по умолчанию указывает на официальный хостед-редактор. То есть `/am
editor` работает сразу после установки.

`config.yml`:

```yaml
editor:
  # Paste-сервис. ПУСТО = общий публичный воркер (ничего настраивать не надо).
  worker-url: ""
  # Адрес хостед-редактора (GitHub Pages). По умолчанию — официальный.
  url: "https://alexfirst404.github.io/alexmenus-editor/"
```

## Self-host воркера (по желанию)

Хочешь свой paste-сервис (приватность/контроль) — разверни воркер из папки
[`cloudflare-worker/`](https://github.com/AlexFirst404/AlexMenus/tree/main/cloudflare-worker) репозитория
(бесплатный tier Cloudflare: ~100k запросов/день, KV бесплатно). Кратко:

1. Аккаунт на Cloudflare, `npm i -g wrangler`, `wrangler login`.
2. `wrangler kv namespace create PASTES`, вставь id в `wrangler.toml`.
3. `wrangler deploy` — получишь адрес вида `https://alexmenus-paste.<акк>.workers.dev`.
4. Впиши его в `config.yml` → `editor.worker-url`, затем `/am reload`.

API воркера: `POST /post` (тело = JSON-бандл) → `{ "key": "<код>" }`; `GET /<код>` → тот же JSON (или `404`,
если истёк). Полная инструкция по деплою — в README папки `cloudflare-worker/`.
