# Начало работы

Эта страница проведёт от установки до первого рабочего меню.

## 1. Структура папок

После первого запуска появляется:

```
plugins/AlexMenus/
├── config.yml            # настройки инвентарь-меню и веб-редактора
└── menus/                # тут лежат все меню
    ├── sample.yml        # пример chest-меню
    └── shop.yml          # пример магазина с requirements
```

Правила загрузки (`MenuStore`):

- **Один YAML-файл = одно меню.** `id` меню — это **имя файла без `.yml`** (например `menus/shop.yml` → id `shop`).
- Папка `menus/` сканируется **рекурсивно**. **Подпапки становятся категориями** (имя категории = путь
  подпапки относительно `menus/`; файлы в корне — категория `general`). Пример: `menus/pvp/kits.yml` → id
  `kits`, категория `pvp`.
- ⚠️ Так как id — это только имя файла, два файла с **одинаковым именем** в разных подпапках дадут
  **конфликт id** (в памяти останется один). Давай файлам уникальные имена.
- Меню перечитываются **на каждом reload** (`/am reload`, `/reload`, рестарт, `/am apply`).

## 2. Первое меню

Создай `plugins/AlexMenus/menus/hello.yml`:

```yaml
type: chest
title: "<gradient:#55ff55:#00aaaa>Привет!</gradient>"
rows: 3
commands: [hello]                       # зарегистрирует команду /hello

items:
  '13':                                 # слот 13 (центр среднего ряда)
    material: DIAMOND
    name: "<aqua>Нажми меня"
    lore:
      - "<gray>ЛКМ — сообщение"
    clicks:
      left:
        - type: message
          text: "<green>Работает!"
  '26':
    material: BARRIER
    name: "<red>Закрыть"
    clicks:
      any:
        - type: close
```

Затем `/am reload`. Открыть можно тремя способами:

- `/hello` — зарегистрированная команда меню;
- `/am open hello` — универсальная команда;
- `/am preview hello` — предпросмотр (только для админа).

Подробнее про поля предмета и клики — [Items-and-Clicks](Items-and-Clicks.md); про заголовок, ряды и
типы — [Menu-Types](Menu-Types.md).

## 3. Reload

`/am reload` (право `alexmenus.admin`) **полностью перезагружает плагин**: перечитывает `config.yml`, все
меню, перерегистрирует команды меню, пересобирает триггеры и веб-редактор. Рестарт не нужен.

## 4. Zero-config веб-редактор (без ручного YAML)

Меню можно собирать мышкой в браузере — порты открывать не надо, работает из коробки:

```
/am editor              # заливает все меню в paste-сервис, выдаёт ссылку <editor>#<код>
   ↓  (правишь в браузере: слоты, предметы, действия, requirements)
/am apply <код>         # скачивает правки, пишет menus/*.yml и перезагружает
```

Подробнее — [Web-Editor](Web-Editor.md). Полная справка по командам и правам — [Commands-and-Permissions](Commands-and-Permissions.md).
