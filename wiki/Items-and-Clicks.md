# Предметы и клики

Предметы задаются под ключом `items`. **Ключ = номер слота** (число в кавычках), значение = описание
предмета.

- Для `chest`-меню слоты: `0 .. rows*9-1`.
- Для `inventory`-меню слоты: `0 .. 26` (ложатся в слоты инвентаря 9–35, см. [Menu-Types](Menu-Types.md)).

Ключ, который не парсится как число, пропускается с предупреждением в лог.

## Поля предмета (`items.<slot>`)

| Поле | Тип | Назначение |
|---|---|---|
| `material` | строка | Материал Bukkit (`DIAMOND`, `LIME_DYE`, …). Неизвестный/не-предмет → `STONE` + предупреждение. |
| `cmd` | строка | Custom-model-data (**строка**, не число) для кастомной текстуры/модели из ресурспака. |
| `name` | строка | Отображаемое имя. Цвета/плейсхолдеры; курсив по умолчанию выключен. |
| `lore` | список строк | Описание. Каждая строка — цвета/плейсхолдеры. |
| `flags` | список строк | Имена `ItemFlag` для скрытия (напр. `HIDE_ATTRIBUTES`, `HIDE_ENCHANTS`). Неизвестные имена пропускаются. |
| `hide-all` | bool | `true` — скрыть **все** item-flags (эквивалент перечисления всех флагов). Имеет приоритет: если `true`, `flags` игнорируется. |
| `clicks` | секция | Действия по типу клика (см. ниже). |
| `view-requirement` | требование | Предмет **виден** только при выполнении условия (см. [Requirements](Requirements.md)). |
| `click-requirement` | требование | **Гейт клика**: не выполнено → `deny`, иначе → `success` + действия клика. |

Полный пример предмета:

```yaml
items:
  '15':
    material: DIAMOND_SWORD
    cmd: "custom_blade"            # строковый custom-model-data
    name: "<red><bold>Меч VIP"
    lore:
      - "<gray>Только для VIP"
      - "<dark_gray>Баланс: <white>%vault_eco_balance%"
    hide-all: true                 # спрятать атрибуты/зачарки/прочий тултип-мусор
    view-requirement:
      type: permission
      permission: alexmenus.vip
    clicks:
      left:
        - type: message
          text: "<red>Взмах!"
```

### Про `cmd` (custom-model-data)

Значение — **строка** (используется строковый компонент `CustomModelDataComponent`), как в 1.21.11.
Ресурспак сопоставляет эту строку своей модели/текстуре. Это НЕ старый целочисленный CMD. Пусто/не задано —
предмет остаётся ванильным.

### Про `flags` / `hide-all`

`flags` — список имён `ItemFlag` (регистр не важен, обрезаются пробелы). `hide-all: true` подставляет **все**
существующие флаги — удобно, чтобы полностью очистить тултип «кнопки». Одно и то же поведение доступно и в
`give_item` (см. [Actions](Actions.md)).

## Клики (`clicks`)

Под `clicks` каждый ключ — тип клика, значение — **список действий** (см. [Actions](Actions.md)). Действия
выполняются по порядку.

```yaml
clicks:
  left:
    - type: message
      text: "<yellow>Левый клик"
  right:
    - type: sound
      sound: "minecraft:block.note_block.bell"
  any:
    - type: refresh
```

### Типы клика

| Ключ в YAML | Алиасы | Bukkit-клики, которые сюда попадают |
|---|---|---|
| `left` | — | `LEFT`, `WINDOW_BORDER_LEFT` |
| `right` | — | `RIGHT`, `WINDOW_BORDER_RIGHT` |
| `shift_left` | `shiftleft` | `SHIFT_LEFT` |
| `shift_right` | `shiftright` | `SHIFT_RIGHT` |
| `middle` | — | `MIDDLE` (клик колесом) |
| `drop` | `q` | `DROP`, `CONTROL_DROP` (клавиша выброса) |
| `any` | `all` | **любой** клик |

**Важно про `any`:** действия из `any` выполняются **вдобавок** к действиям конкретного типа. Порядок:
сначала действия конкретного клика (например `left`), затем `any`. То есть `any` — это «общий хвост» для
всех кликов, а не «fallback только когда нет конкретного».

```yaml
clicks:
  left:
    - type: message
      text: "Только по ЛКМ"
  any:
    - type: sound             # прозвучит и по ЛКМ, и по любому другому клику
      sound: "minecraft:ui.button.click"
```

Клики, которые ни во что не маппятся, считаются `UNKNOWN` — по ним срабатывает **только** `any`.

### Порядок обработки клика

1. Если у предмета есть `click-requirement` и условие **не** выполнено → выполняются `deny`-действия,
   клик **останавливается**.
2. Если условие выполнено → выполняются `success`-действия (если заданы), затем действия клика
   (конкретный тип + `any`).
3. Предмет со **скрытым** `view-requirement` не отрисовывается и клики по нему не срабатывают (в обоих
   бэкендах пустая ячейка инертна).

Подробности по условиям — [Requirements](Requirements.md); по действиям — [Actions](Actions.md).
